Help about Classification Upload Tool

Upload File Format:

The upload file should be of CSV (Comma Separated Values) format. To get this format from an Excel Spread Sheet, follow these steps-

  1. Click on Save As
  2. Select 'Save as type:' as 'CSV (Comma delimited) (*.csv)
  3. Click on Save.

To check this, you can open this file using a "notepad" or "WordPad", and it will look like this, with values separated in a ',' (comma)-

GDW,20102099-00001-A0,SDHC EMPLOYEE ACCT,CUSTOMER_CATEGORY,HIGHER EDUCATION,,Y,XXGDW
GDW,20128210-00001-A0,BALDWIN CNTY BOARD OF ED,CUSTOMER_CATEGORY,HIGHER EDUCATION,,Y,XXGDW
GDW,20131405-00001-A0,UNIV OF WY-MOLECULAR BIO,CUSTOMER_CATEGORY,HIGHER EDUCATION,,Y,XXGDW

Upload File Values:

The upload file should contain the following values in the following order-

  1. Party Orig System
  2. Party Orig System Reference
  3. Party Name
  4. Class Category
  5. Class Code
  6. Start Date Active
  7. Primary Flag
  8. Created By Module

Please validate for Some of the columns as follows:

  1. Class_code:

    2. - Please note that the class_code should be in the upload file. Keeping a class_code meaning or description in place of class_code is invalid, and the upload will fail.

    - The Class_category for which the class_codes are assigned must be valid one, which appears in OCO UI.

  2. Primary_Flag:

    3. - The primary_flag should be either .Y. or .N.. Keeping .Yes. and .No. or other values is invalid, and upload fails.

    - For a party, and for a class_category, only one class_code can be primary. So, when uploading a file, unless you are sure about it, please make the primary_flag to .N., so that the upload will be successful.

    - If there exist a primary class_code for a given class_category and user tries to upload a class_code with primary_flag as .Y., then, the existing primary code assignment will be made .N. automatically and the new one will be made primary.

    - There can be other situation, where in a primary code assignment was end dated; And user tries to upload the same code. This situation is called, .Restoring.. In this situation, if there is already another .active. primary code existing, and user tries to restore the end_dated code assignment (which was primary), application throws error for this record, as "You cannot set this class code as primary because its date range overlaps with an existing primary class code in the same class category."

Errors, "What it Means?":
  1. You cannot set this class code as primary because its date range overlaps with an existing primary class code in the same class category.

    2. Reason: If there is already another .active. primary code existing, and user tries to restore the end dated code assignment (which was primary), application throws this error for this record.

  2. End date cannot be earlier than the start date.

    3. Reason: Self explanatory.

  3. The table name or primary key is either not valid or not eligible to be classified by this class category.

    4. Reason: Classifications can only be assigned to Organization type parties.
