Customer Profiles Upload Tool

 

This is a self service application, where user can upload customer profiles like Collector, Payment Terms, Dunning information, Consolidated Billing information etc.

Steps in Uploading Customer Profiles:


1. Prepare an input file, which is of CSV format (Comma Separated Format). This format can be obtained like this-
   a).Open an Excel Spread sheet
   b).Add all the data in the format mention in the instructions-
   ----First Column - Payment Term
   ----Second Column - Credit Checking Flag
   ----Third Column - Dunning Letters Flag
   ----Forth Column - Dunning Letters Set
   ----Fifth Column - Credit Hold Flag
   ----Sixth Column - Consolidated Bill Flag
   ----Seventh Column - Consolidated Bill Type
   ----Eighth Column - Oracle Customer Account Number (xxxxxxxx-00001-A0) (the first 8 characters are AOPS customer IDs left padded with zeros)

Note that if any of the above columns are null, please leave the value as null. BUT, if you leave the last column also null, then Microsoft Excel does not put the commas as desired and the file format may be wrong and you may get errors in the file upload. After the data is added in Excel spread sheet, save it as "Comma Separated Values" format.

So, a typical upload file, which is in CSV format, when opened in a notepad, looks like this- 

 
 
Collector,Pay Terms,Credit ck flag on,Dunning Send letters flag,Dunning set,Credit hold flag,Cons_Inv_Flag,Cons_inv_Type,Oracle account #
,,,,Y,Y,,1460
"NET 30",Y,Y,STANDARD_OD,N,Y,,1357
"NET 45",Y,Y,STANDARD_OD,N,Y,,1460
"NET 60",Y,Y,STANDARD_OD,N,Y,,99898
 

2. Select the file from the local file system, using "Browse" button in the page.

3. Click on "Upload Customer Profiles" Button in the page.

4. That's it!

This file will be uploaded into server. A concurrent request set will be submitted in the background to synch the customer profiles into CDH registry (TCA registry). The tool shows a confirmation message telling that the file has been uploaded into the server; And a request set, has been submitted in the back ground. Please check the concurrent request status using standard fnd concurrent request screen for the completion.
